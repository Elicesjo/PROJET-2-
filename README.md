# Requêtes SQL — Marchand de vin

> 10+ requêtes SQL complexes (jointures multi-tables, agrégats, sous-requêtes) pour analyser les ventes et identifier les produits et clients les plus rentables.

**Stack** · SQL · SQLite · DBeaver

---

## Contexte

Base de données relationnelle d'un marchand de vin (Rester dans les Vignes). À partir du schéma entité-relation fourni, l'objectif est d'explorer les données de vente et d'écrire des requêtes progressivement complexes pour répondre à des questions métier précises.

## Requêtes réalisées

| Catégorie | Exemples |
|-----------|---------|
| Sélection simple | Produits par région, clients par ville |
| Jointures | Commandes avec détail produit et client |
| Agrégations | CA par produit, panier moyen, top 5 clients |
| Sous-requêtes | Produits au-dessus de la moyenne, clients sans commande |
| Filtres avancés | Périodes, seuils, conditions multiples |

## Résultats clés

- Identification des 5 produits les plus rentables par région viticole
- Détection des clients à fort potentiel (RFM simplifié)
- Analyse de la saisonnalité des ventes sur 2 ans
